# 18 人物采访式共创方案

## 1. 文档目的

本文档用于回答一个核心问题：

当前人物讨论为什么像问卷调查，以及如何把它改造成“像朋友聊天”或“像专业采访记者采访”的共创体验。

目标不是放弃结构化，而是把结构化隐藏到后台，把前台体验改成自然、轻松、能激发创意的采访式对话。

## 2. 当前问题判断

当前人物讨论之所以显得死板，根因有 3 个：

1. 前台是显式题库驱动  
   系统按固定维度逐题推进，用户感受到的是“系统在问我要标准答案”。

2. 回答被强行归入单一问题  
   作者说一段复杂描述，系统倾向于把它塞进当前题目，而不是同时吸收多种人物信息。

3. 系统更像验收，不像采访  
   当前机制重“补齐字段”，轻“接住感觉、复述理解、顺势追问、激发灵感”。

## 3. 目标体验

改造后的人物讨论应接近两种风格：

### 3.1 朋友式共创

- 先接住作者脑海里的感觉、画面、关系火花
- 不急着定义专业术语
- 允许模糊表达、半成品表达、临时灵感

### 3.2 记者式采访

- 会先复述“我目前理解到的是……”
- 会围绕矛盾、反差、关系、动机做顺势追问
- 会问出关键点，但不会让作者感到被盘问

## 4. 设计原则

### P1. 作者说故事，不是填答案

系统默认接受段落式表达、场景式表达、情绪式表达，而不是优先要求用户命中预设选项。

### P2. 系统做采访，不做阅卷

每轮优先做四件事：

1. 接住
2. 复述
3. 提炼
4. 追问

### P3. 字段是结果，不是入口

人物卡字段继续保留，但字段应由自由描述与讨论结果逐步回填，而不是作为对话的起点。

### P4. 一次回答可以命中多个维度

作者一句话里可能同时包含：

- 第一印象
- 欲望
- 恐惧
- 关系张力
- 行动方式

系统应允许一段内容同时更新多个维度，而不是只归类到当前一道题。

### P5. 追问围绕“最有戏的缺口”

不按固定题目顺序问，而是优先追当前最值得继续深挖的那个点，例如：

- 反差
- 矛盾
- 创伤来源
- 关系火花
- 决策底层逻辑

## 5. 目标交互形态

人物设定页中的对话区，每轮应只展示：

1. 系统当前理解
2. 当前最值得继续追问的一点
3. 一个自然追问
4. 仅在用户卡住时出现的灵感辅助按钮

示例：

> 我先确认一下，我现在理解到的是：她表面冷静克制，但真正驱动她的是摆脱被家族当成交换筹码。  
> 我现在最想继续追的是：她为什么明知道男主危险，还是会继续靠近他？  
> 这个靠近，更多是利益需要，还是她在他身上看到了自己想逃离的命运？

这种形式比“请选择她的关键关系张力”更接近真实采访。

## 6. 系统能力改造

### 6.1 从题库驱动改成采访策略驱动

保留后台维度，但前台不再直接暴露为固定问卷。

后台继续维护：

- 第一印象
- 核心欲望
- 核心恐惧
- 关键关系张力
- 行动方式 / 决策模式
- 人物边界
- 成长缺口
- 伪装与真实自我

但系统每轮只输出“当前最值得追的一问”。

### 6.2 从单题归类改成多维提取

用户一段回答应支持：

- 多维同时命中
- 每维记录置信度
- 区分“暂定理解”和“已确认理解”

### 6.3 增加采访人格层

建议支持 3 种模式：

1. 轻松共创  
   更像朋友聊天，适合刚起步的作者

2. 编辑式深挖  
   更擅长抓矛盾、抓反差、抓结构缺口

3. 记者式采访  
   更擅长追问动机、关系、事件与人物逻辑

### 6.4 增加灵感激发动作

当作者卡住时，系统不只给选项，还应提供创意型辅助：

- 给我一个更狠的版本
- 给我一个更暧昧的关系版本
- 给我一个更有反差的版本
- 用一个场景帮我试出来
- 用一句台词试出她的气质

## 7. 页面改造建议

人物设定页对话区建议拆成三层：

### 第一层：当前理解

- 系统目前理解到的人物草稿
- 标记哪些是暂定、哪些已确认

### 第二层：当前追问

- 只展示一个当前主问题
- 用自然语言提问
- 不默认展示大量预设选项

### 第三层：灵感辅助

- 仅在用户主动点击或系统判断卡住时显示
- 提供“换个问法 / 给我例子 / 给我更狠版本 / 用场景试一下”

## 8. 实施顺序

建议按以下顺序落地：

1. 保留后台人物维度，但移除前台默认问卷感
2. 把每轮输出改成“理解 + 追问 + 辅助”
3. 把回答解析改成“多维提取 + 置信度”
4. 增加采访人格和灵感激发动作
5. 最后再优化草稿确认、人物摘要、人物画像的联动

## 9. 验收标准

改造完成后，应满足：

1. 作者可以连续用自然段描述，不会被频繁拉回预设题目
2. 系统每轮提问更像顺着作者的话继续聊，而不是切题盘问
3. 一次回答可以更新多项人物理解
4. 当作者卡住时，系统能提供激发创意的辅助，而不是重复同一题
5. 作者主观感受应更接近“被采访、被启发、被一起整理”，而不是“被问卷收集”

## 10. 结论

可以达到你期待的效果，但关键不是把问题写得更口语一点，而是把整套人物讨论机制从“问卷模式”改成“采访模式”。

一句话概括：

**前台像聊天，后台像编辑。**

只有这样，系统既不会限制作者创意，又能在轻松交谈中把真正关键的人物信息收出来。
