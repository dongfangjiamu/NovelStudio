# 16 人物设定共创与人物画像方案

## 背景问题

当前人物设定页偏向“表单录入”，更像把人物属性填进去，但缺少系统引导讨论的过程。对没有经验的作者来说，这会带来两个直接问题：

1. 不知道人物设定应该讨论到什么程度才算够。
2. 虽然填了字段，但没有经过讨论收束，无法沉淀成稳定可写的人物画像。

因此，人物设定不应只有人物卡表单，还应具备“引导讨论”和“画像生成”两层能力。

## 目标

围绕每一张人物卡，形成完整闭环：

1. 先填写人物卡基础信息。
2. 系统提示当前缺口，逐步引导讨论。
3. 结合表单字段与讨论结论，自动生成人物画像。
4. 让后续章卡、正文、审查都能复用这份画像。

## 产品结构

人物设定页按“单张人物卡”组织，每次只处理当前选中的人物卡。

### 1. 人物卡基础字段

用于录入作者已经明确的信息：

- 人物卡标签
- 角色名
- 主次定位
- 剧情职责
- 与主角关系
- 核心欲望
- 主要恐惧
- 声音与气质
- 补充备注

### 2. 系统引导讨论

这是人物设定的核心层。作者选中一张人物卡后，系统不应只展示表单，而应明确提示：

- 这张卡当前还缺什么
- 建议先讨论哪一项
- 已经确认了什么
- 还差多少才能形成稳定人物设定

讨论不是自由发散，而是按维度收束。

### 3. 自动生成人物画像

当当前人物卡的基础字段与讨论结论达到一定完整度后，系统提供“生成人物画像”或“基于最新讨论重新生成画像”按钮，输出后续写作可直接复用的稳定设定稿。

## 每张人物卡最低讨论维度

建议每张人物卡至少收束以下 6 个维度：

1. 第一印象
2. 核心欲望
3. 核心恐惧
4. 关键关系张力
5. 行动方式 / 决策模式
6. 人物边界

如果是主角，再增加：

1. 成长缺口
2. 伪装与真实自我

系统需要给出完成度提示，例如：`当前已完成 4/6，建议先补：关键关系张力、人物边界。`

## 人物画像输出结构

人物画像不应只是摘要，而应是稳定的写作基底，建议包含：

1. 人物定位
2. 外在气质
3. 内在驱动力
4. 核心矛盾
5. 关键关系张力
6. 行动逻辑
7. 情绪触发点
8. 绝不能写崩的边界
9. 典型说话方式
10. 当前剧情阶段状态

## 页面交互建议

选中某张人物卡后，页面展示三块内容：

1. 基础字段区
2. 引导讨论区
3. 人物画像区

引导讨论区负责告诉作者“先补什么”；人物画像区负责把“字段 + 讨论”融合成可复用设定。

## 核心原则

不要让作者自己判断“设定够不够”。  
应该由系统明确告诉作者：

- 现在还缺什么
- 为什么缺这个
- 先补哪个最划算
- 达到什么程度就可以生成稳定画像

## 落地顺序

1. 恢复人物设定页中的引导讨论区
2. 为每张人物卡增加完成度与缺口提示
3. 增加“生成人物画像”面板
4. 用“人物字段 + 讨论结论”联合生成画像，而不是只依赖表单
