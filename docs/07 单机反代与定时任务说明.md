# NovelStudio 单机反代与定时任务说明

本文档面向当前 VPS 的内部部署场景。

## 1. 默认网络策略

根目录 `docker-compose.yml` 当前默认把应用只绑定到本机回环地址：

```bash
APP_BIND_HOST=127.0.0.1
APP_PORT=18080
```

这意味着：

- 应用默认不会直接暴露到公网
- 本机可通过 `127.0.0.1:18080` 访问
- 后续若要接反向代理，应让代理转发到这个本地端口

## 2. 反向代理样例

仓库已提供本地 Caddy 样例：

- `deploy/caddy/Caddyfile.local`

当前样例只监听本机 `127.0.0.1:8081`，并转发到 `127.0.0.1:18080`，用于本地联调，不会直接对外暴露。

如果后续要真正公网访问，再额外补：

- 域名
- HTTPS
- 基础访问控制

## 3. 定时备份

仓库已提供：

- `ops/run_backup_cycle.sh`
- `ops/prune_backups.sh`
- `ops/install_backup_cron.sh`

默认备份周期脚本会执行：

1. 创建 Postgres 逻辑备份
2. 按保留策略清理历史备份

保留策略默认值：

- 最近 `7` 天按天保留
- 更早数据按周保留最近 `4` 周

## 4. 查看 cron 内容

先打印计划任务，不直接安装：

```bash
make print-backup-cron
```

默认输出相当于：

```bash
17 3 * * * root cd /root/NovelStudio && ENV_FILE=/root/NovelStudio/.env.compose COMPOSE_PROJECT_NAME= /root/NovelStudio/ops/run_backup_cycle.sh >> /var/log/novelstudio-backup.log 2>&1
```

## 5. 安装 cron

如果确认要启用：

```bash
ENV_FILE=/root/NovelStudio/.env.compose /root/NovelStudio/ops/install_backup_cron.sh
```

默认会写入：

- `/etc/cron.d/novelstudio-backup`

## 6. 手动执行

手动跑一轮：

```bash
make backup-cycle
```

只清理历史备份：

```bash
make prune-backups
```

只查看将删除哪些文件：

```bash
DRY_RUN=true /root/NovelStudio/ops/prune_backups.sh
```

## 7. 当前边界

- 当前只做了本机回环绑定和本地反代样例
- 当前没有自动申请证书
- 当前没有防火墙规则收口说明
- 如果后续改成公网访问，需要再补 HTTPS、访问控制和暴露面审查
