# 13 立项引导与意图澄清方案

## 1. 文档目的

本文档用于把 NovelStudio 的“前期共创采访模式”收口成一套正式、可实施、可验收的产品与交互设计。

它要解决的不是“把现有表单问题改简单一点”，而是更根本的问题：

- 创作者在开书最早期往往没有结构化答案
- 创作者经常只有一个片段、一个人物感觉、一个场景、一个情绪
- 系统不应该先要求创作者回答“核心钩子”“核心爽点”“平台定位”这类编辑语言
- 系统应该先帮助创作者捕获、澄清、归纳、确认自己的模糊意图

因此，本方案的目标是让 NovelStudio 从“表单式信息收集器”升级为“创作意图澄清器”。

## 2. 问题定义

### 2.1 当前入口的主要问题

当前“新建项目”页默认要求用户填写：

- 项目名
- 作品标题
- 题材
- 平台
- 核心钩子
- 必须包含
- 必须避免

这套输入结构适合已经把书想清楚的用户，但不适合真实的大多数创作起点。

### 2.2 用户真实状态

创作者刚开书时更常见的状态是：

- 只有一个画面
- 只有一个人物印象
- 只有一个“如果……会怎样”的设定
- 只有“我想写这种感觉”的模糊冲动
- 能判断“更像 A 还是 B”，但很难直接写出完整创作 brief

### 2.3 真实需求

系统真正应该做的不是“等待填写”，而是：

1. 接住模糊灵感
2. 用低压力的小问题逐步追问
3. 把模糊表达翻译成结构化理解
4. 定期反述当前理解并让创作者确认
5. 最终沉淀成后续写作可执行的文本

## 3. 目标与边界

## 3.1 产品目标

### G1. 降低开书门槛

让创作者只凭一句话、一个片段或一个模糊感觉，也能开始立项。

### G2. 用对话逐步收敛意图

系统不再一次性索要完整信息，而是通过若干轮追问逐渐逼近创作者真正想写的东西。

### G3. 自动整理成可确认草案

系统要把创作者的碎片表达整理成：

- 项目方向草案
- 人物设定草案
- 卷纲方向草案
- 长期写作规则草案

### G4. 把结论接入后续流程

澄清阶段形成的稳定结论必须能直接进入：

- `default_user_brief`
- `character_note`
- `outline_constraint`
- `writer_playbook_rule`

## 3.2 本阶段边界

本阶段不做：

- 复杂人格测评
- 自动心理画像
- 高自由度智能问卷编排引擎
- 完整卷纲自动生成器

本阶段只做：

- 低门槛入口
- 渐进式引导问题
- 候选答案与短回答结合
- 周期性反述与确认
- 结构化沉淀

## 4. 设计原则

### P1. 先接住，再澄清

第一步不是让用户把话说清，而是先把用户脑中的模糊念头接住。

### P2. 多选少写

开书早期优先使用：

- 二选一
- 三选一
- 多选
- 句子补全
- 一两句话短回答

避免一开始就要求长文本输入。

### P3. “不知道”是合法答案

系统必须允许这些反馈：

- 我不确定
- 给我几个选项
- 这几个都不对
- 先跳过

### P4. 每 2 到 3 轮就反述一次理解

系统不能只追问，必须定期总结：

- 我现在理解你想写的是……
- 目前已确认的是……
- 还模糊的是……
- 下一步只需要再确认这一个点

### P5. 结论必须可采纳

对话不是目的。对话中形成的稳定结论必须能被采纳为后续工作流输入。

## 5. 推荐交互形态

## 5.1 入口形态

将当前“新建项目”默认入口拆成两层：

### 入口 A：从灵感开始

适合还没有想清楚的创作者。

用户只需提供：

- 一个片段
- 一个角色感觉
- 一个场景
- 一句设定
- 一个参考作品印象

### 入口 B：我已经想清楚了

保留现有结构化表单与高级 JSON，供高成熟度用户直接使用。

## 5.2 工作流形态

推荐使用“灵感捕获 -> 引导对话 -> 草案确认”的三段式主路径：

1. 灵感捕获
2. 引导澄清
3. 草案确认并进入正式创作

## 6. 最小可落地方案

## 6.1 阶段 1：灵感捕获入口

### 输入项

- 项目代号或暂定书名
- 你现在脑子里最清楚的一点点想法
- 这更像什么：
  - 一个画面
  - 一个角色
  - 一个冲突
  - 一个设定
  - 一种氛围
  - 一个参考作品印象

### 产出

创建项目，并自动生成一条 `立项共创` 线程。

项目基础 brief 先只保存：

- `title`
- `idea_seed`
- `idea_seed_type`
- `capture_stage = "seed"`

## 6.2 阶段 2：低门槛引导问答

### 第一批要问的不是“核心爽点”

而是更容易回答的问题，例如：

1. 你最想保住的是什么？
   - 爽感
   - 情绪
   - 悬念
   - 人物关系
   - 我还说不清

2. 主角更像哪一种？
   - 主动争
   - 被迫卷入
   - 隐忍反击
   - 我还不确定

3. 你更想先强调哪种推进？
   - 升级
   - 阴谋
   - 关系
   - 生存
   - 还没想清

4. 这本书最不想丢掉的感觉是什么？
   - 热血
   - 压迫
   - 神秘
   - 上头
   - 轻快
   - 其他

### 输出方式

每轮系统输出：

- 当前已确认
- 还未确认
- 下一问
- 可直接点选的 3 到 5 个选项

## 6.3 阶段 3：草案反述

当信息足够时，系统自动整理一版“当前理解”：

- 这是一个怎样的故事
- 主角当前大致是什么样的人
- 故事更偏什么驱动
- 读者最可能被什么吸引
- 还有哪 2 到 3 个关键未决问题

用户可选择：

- 继续追问
- 采纳部分结论
- 进入人物讨论
- 进入大纲讨论

## 7. 信息模型

建议在 `default_user_brief` 内增加一批早期立项字段：

```json
{
  "title": "暂定书名",
  "idea_seed": "一个被逐出山门的外门弟子，靠偷听禁地炉火里的古老对话逆天改命。",
  "idea_seed_type": "setup",
  "capture_stage": "seed",
  "intent_profile": {
    "reader_pull": "suspense",
    "protagonist_mode": "restrained_counterattack",
    "drive_mode": "conspiracy",
    "core_feeling": "压迫中翻盘"
  }
}
```

建议在采访状态中显式维护：

- `goal`
- `completion_count`
- `total_topics`
- `confirmed_topics`
- `unresolved_topics`
- `next_prompt`
- `next_options`
- `answer_mode`
- `reflection_summary`

## 8. 推荐问题编排

## 8.1 项目级立项共创

建议问题顺序：

1. 先确认最想保住的吸引力
2. 再确认主角行动方式
3. 再确认故事推进方式
4. 再确认不能写歪的边界

而不是反过来。

## 8.2 人物讨论

建议从这些问题入手：

- 主角更像什么类型的人
- 他真正想摆脱什么
- 他最怕失去什么
- 什么行为一出现就会写崩

## 8.3 大纲讨论

建议从这些问题入手：

- 第一卷到底靠什么往前推
- 中段最重要的变化是什么
- 卷末必须兑现什么

## 9. 前端实现建议

## 9.1 项目设定页

将当前单一表单改成两块：

### 模块 A：灵感捕获

- 暂定书名
- 一点点想法
- 灵感类型选择
- “开始共创澄清”按钮

### 模块 B：高级创建

折叠保留原有结构化表单。

## 9.2 对话协作页

在 `立项共创 / 人物讨论 / 大纲讨论` 线程里新增：

- 当前采访阶段说明
- 已确认事项
- 仍待澄清事项
- 下一问
- 候选答案按钮
- “我不确定 / 跳过 / 换个问法”按钮

## 10. 后端实现建议

## 10.1 线程开场

`project_bootstrap` 开场不再直接问“核心爽点是什么”，而是：

- 先反述用户给出的灵感碎片
- 再给出一条低门槛问题
- 同时附上 3 到 5 个可直接选择的答案

## 10.2 跟进消息

每轮用户回答后，系统应返回：

- 已记录的理解
- 当前进度
- 已确认事项
- 下一问
- 候选答案

## 10.3 结构化结论

当用户在低门槛对话中形成稳定偏好时，应支持采纳为：

- `character_note`
- `outline_constraint`
- `writer_playbook_rule`

必要时也可同步写回 `default_user_brief.intent_profile`

## 11. 验收标准

本阶段完成的最低标准是：

1. 创作者可以只靠一句模糊灵感创建项目
2. 新项目默认进入 `立项共创`，而不是要求先填完整表单
3. 立项线程里的问题不再以编辑术语为主
4. 至少支持 3 轮低门槛问题推进
5. 系统会输出“已确认 / 未决 / 下一问”
6. 至少一部分回答支持点选而不是必须自由输入
7. 形成的结论能进入后续运行

## 12. 路线建议

### Phase 1

- 新建项目默认改成“灵感捕获入口”
- 保留高级表单
- 立项共创问题改为低门槛版本
- 支持候选答案按钮

### Phase 2

- 增加分支式问题选择
- 增加“换个问法 / 我不确定 / 先跳过”
- 增加中途反述草案

### Phase 3

- 让人物讨论和大纲讨论也进入同样的引导式模式
- 支持候选方案对比
- 支持“更商业 / 更情绪 / 更悬疑”版本切换

## 13. 最终结论

NovelStudio 的立项入口不应默认是“让用户填写一套已经想清楚的创作 brief”。

更合理的路径是：

- 从模糊灵感开始
- 用低压力问题逐步澄清
- 用系统反述不断校准理解
- 最终再把结论沉淀成可执行文本

也就是说，系统应先成为：

`创作意图澄清器`

然后才是：

`写作流程执行器`
