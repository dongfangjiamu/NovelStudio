# NovelStudio 单机自恢复与巡检说明

本文档说明当前 VPS 上的开机自恢复与最小巡检骨架。

## 1. systemd 单元文件

仓库已提供以下模板：

- `deploy/systemd/novelstudio-compose.service`
- `deploy/systemd/novelstudio-backup.service`
- `deploy/systemd/novelstudio-backup.timer`
- `deploy/systemd/novelstudio-healthcheck.service`
- `deploy/systemd/novelstudio-healthcheck.timer`

目标：

- 机器启动后自动拉起 Compose 栈
- 每晚自动执行备份周期
- 每 5 分钟执行一次最小健康巡检

## 2. 巡检脚本

巡检脚本：

- `ops/check_system_health.sh`

当前会检查：

- `APP_URL` 对应的 `/healthz`
- 数据库状态是否为 `ready`
- 磁盘占用是否超过阈值
- 内存占用是否超过阈值
- 1 分钟负载是否超过按 CPU 数量放大的阈值

默认阈值：

- `DISK_WARN_PERCENT=85`
- `MEMORY_WARN_PERCENT=90`
- `LOAD_WARN_PER_CPU=1.5`

手动执行：

```bash
make health-check
```

## 3. 验证 unit 文件

先只做语法和依赖验证：

```bash
make systemd-verify
```

## 4. 安装 unit 文件

如果确认要启用：

```bash
/root/NovelStudio/ops/install_systemd_units.sh
```

如果要安装后立即启用：

```bash
/root/NovelStudio/ops/install_systemd_units.sh --enable
```

默认会安装到：

- `/etc/systemd/system`

## 5. 建议启用顺序

建议顺序：

1. 先确认 `.env.compose` 已配置
2. 执行 `make systemd-verify`
3. 执行 `/root/NovelStudio/ops/install_systemd_units.sh`
4. 手动运行 `systemctl start novelstudio-compose.service`
5. 确认 `make health-check` 通过
6. 再决定是否启用 backup / healthcheck timer

## 6. 当前边界

- 当前巡检脚本只做本机最小检查，没有外部告警发送
- 当前 systemd 文件假定仓库路径是 `/root/NovelStudio`
- 如果后续迁移目录，需要同步修改 unit 文件中的绝对路径
