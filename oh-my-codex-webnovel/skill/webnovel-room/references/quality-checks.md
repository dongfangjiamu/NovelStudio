# 质量检查清单

## 一章发布前的硬性检查

1. **开头 300 字内有钩子**：问题、危机、冲突、反常信息至少占一项。
2. **本章有目标**：主角想做什么，读者看得懂。
3. **有阻力**：不是自动推进，而是被对抗、被误导、被付费。
4. **有变化**：状态、关系、信息、筹码至少变化一项。
5. **有成本**：赢也要付代价，输要带来后续麻烦。
6. **尾钩可追更**：结尾留下明确的未完成张力。
7. **不崩设**：规则、战力、伤势、位置、时间都对得上。

## 四项核心评分

### 爽点
- 5：期待被兑现且超出预期
- 4：兑现明确，满足感够
- 3：有兑现但偏平
- 2：承诺没兑现或被解释稀释
- 1：无明显爽点

### 节奏
- 5：信息、动作、情绪切换紧凑，无明显灌水
- 4：整体顺，局部略慢
- 3：中段拖沓
- 2：大量解释或重复总结
- 1：读者容易弃读

### 代入
- 5：主角欲望清楚，情绪和选择可跟随
- 4：主要段落可代入
- 3：偶尔抽离
- 2：像旁白总结
- 1：人物像设定说明书

### 追更欲
- 5：结尾制造强烈下一章需求
- 4：愿意继续看
- 3：可追可不追
- 2：收尾太平
- 1：没有悬念

## 常见问题与优先修法

- **说明腔过重** -> 砍掉结论句，改成动作、对话、反应展示。
- **章节像过场** -> 给本章一个必须完成却难以完成的小目标。
- **主角被动** -> 增加主动选择，哪怕选择错误。
- **冲突不够尖** -> 提高代价或缩短时限。
- **尾钩无力** -> 让结尾落在“新问题”而不是“总结感受”。
- **AI 味** -> 删除同义反复、空泛判断句、无必要修辞堆叠。
