# Webnovel Room 安装与使用说明

## 你拿到的两个包

1. `skill.zip`
   - 用途：上传到 ChatGPT Skills，作为可自动调用的多角色网文写作技能。
2. `webnovel-project-template.zip`
   - 用途：不用 Skills 也能直接用，适合放进 ChatGPT Projects 里当长期项目模板。

---

## 方案 A：安装为 ChatGPT Skill（推荐）

### 适用场景
- 你所在账号或工作区已经开通 Skills。
- 你希望 ChatGPT 在你提到“做卷纲、做章节卡、续写、审校”等任务时自动识别并调用这一套流程。

### 安装步骤
1. 打开 ChatGPT 网页端。
2. 点击左下角头像。
3. 进入 **Skills**。
4. 点击 **New skill**。
5. 选择 **Upload from your computer**。
6. 上传 `skill.zip`。
7. 安装后，在新对话中直接说：
   - 帮我立项一本男频都市异能网文
   - 先给我做世界观 bible 和人物卡
   - 基于这套设定继续写下一章，但先做章节卡
   - 检查这 10 章有没有设定冲突

### 推荐首轮配置
把下面四项在第一次对话里补齐：
- 类型 / 子类
- 目标读者
- 三个核心卖点
- 20 章内必须兑现的承诺

### 推荐开场提示词
```text
你现在作为 Webnovel Room 工作。先不要直接写正文。
请按以下顺序输出：
1. 00 项目简报
2. 01 类型定位
3. 02 世界观 bible
4. 03 人物卡
在信息不足时允许你做“默认设定”，但请显式标注“默认”。
题材：都市异能
目标读者：男频成长流
核心卖点：底层逆袭、组织斗争、能力升级
```

## 方案 B：不用 Skills，直接放进 ChatGPT Projects

### 适用场景
- 你的账号暂时没有 Skills 入口。
- 你想立刻开始用，并希望长期保存设定、章节卡和时间线。

### 安装步骤
1. 打开 ChatGPT。
2. 新建一个 **Project**，比如命名为“我的网文项目”。
3. 解压 `webnovel-project-template.zip`。
4. 把解压后的所有文件上传到这个 Project。
5. 在 Project 的 instructions 中粘贴下面这段：

```text
你是我的网文编辑部系统。
默认按“主编 -> 设定官 -> 人设官 -> 剧情架构师 -> 章节编剧 -> 正文作者 -> 连贯性审校 -> 读者模拟器 -> 风格编辑”的顺序工作。
先做上游文档，再写正文；先给章节卡，再写章节。
所有重要改动都要回写到以下文件：
- 02-world-bible.md
- 03-character-cards.md
- 05-foreshadow-ledger.md
- 06-timeline.md
如果信息不足，可以做默认设定，但必须标记为“默认”。
默认输出中文，目标是高留存、高追更欲、低崩设率。
```

### 第一次进入项目时这样说
```text
请读取项目中的 README.md 和 00-10 文件模板。
先帮我完成 00 项目简报和 01 类型定位。
题材是：
目标读者是：
三个核心卖点是：
我不想写的内容是：
```

## 标准工作流

### 新书立项
1. 完成 00 项目简报
2. 完成 01 类型定位
3. 完成 02 世界观 Bible
4. 完成 03 人物卡
5. 完成 04 第一卷卷纲
6. 建立 05 伏笔账本
7. 建立 06 时间线
8. 为下一章生成 07 章节卡
9. 根据章节卡写正文到 08
10. 按 09 做修订检查

### 日常连载
每次写新章节前：
1. 读取 02、03、04、05、06
2. 只做下一章的 07 章节卡
3. 确认尾钩类型
4. 写正文
5. 更新 05 和 06

### 修文与诊断
直接说：
```text
请按“症状 -> 原因 -> 最小修法 -> 结构性修法”给我诊断这一章。
重点检查：节奏、爽点、追更欲、设定连贯性。
```

## 常用指令

### 1. 立项
```text
基于这个项目包，先帮我完成 00 和 01。
题材：女频仙侠
目标读者：偏情绪流、成长流
三个核心卖点：高压关系、身份反转、命格升级
```

### 2. 做第一卷卷纲
```text
基于 00-03，帮我规划第一卷 20 章卷纲，写入 04。
要求每 3-5 章都有一次明显升级或反转。
```

### 3. 做下一章章节卡
```text
读取 02-06，给我第 8 章的章节卡。
本章必须推进主线，不能只解释设定。
结尾要有强追更钩子。
```

### 4. 起草正文
```text
按 chapters/ch008-card.md 的思路写 2800 字正文。
要求场景化，不要总结腔，不要 AI 味。
```

### 5. 审校
```text
按 09 修订清单审校这章。
先给问题单，再给修订版。
```

## 目录说明

- `00-project-brief.md`：立项简报
- `01-genre-positioning.md`：类型定位和产品化拆解
- `02-world-bible.md`：世界规则、势力、地点、代价
- `03-character-cards.md`：人物动机、关系、成长弧线
- `04-arc-outline.md`：分卷大纲
- `05-foreshadow-ledger.md`：伏笔账本
- `06-timeline.md`：时间线
- `07-chapter-card.md`：章节卡模板
- `08-draft.md`：正文草稿区
- `09-revision-checklist.md`：审校清单
- `10-project-config.yml`：流程和质量阈值配置

---

## 建议的使用纪律

- 不要一上来连写 20 章，先把 00-06 稳住。
- 每次只写一章或一小段，写完就更新时间线和伏笔账本。
- 如果要大改设定，先改 02 和 03，再改后续正文。
- 如果某章很无聊，优先检查“本章目标、阻力、代价、尾钩”四项。
